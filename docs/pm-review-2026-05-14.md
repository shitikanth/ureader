# ureader PM Review — 2026-05-14

## What ships today

A functional epub reader for macOS. Users open books via Finder ("Open With" or file association), read chapter by chapter with a collapsible TOC sidebar, and return to their last position on re-open. The renderer is native WKWebView so most epub content renders correctly.

---

## User experience gaps

### Opening books
The app exits silently if launched without a file. There is no way to open a different book from within a window — the user must use Finder again. There is no recent files list.

**Impact:** Medium. Power users quickly learn the workflow, but new users will be confused by an app that appears to do nothing when launched from the Dock.

---

### Reading comfort
The shell UI is dark but the epub content renders in the iframe with the book's own CSS — typically white background. The result is a visually jarring light rectangle inside a dark chrome.

There are no typography controls: no font size adjustment, no line spacing, no font choice, no margins. No theme switching (light / dark / sepia).

**Impact:** High. These are baseline expectations for any reading app. Users with vision needs will find the app unusable.

---

### Navigation
Keyboard navigation is not implemented. Users cannot press arrow keys, space, or page-up/down to move through a book.

There is no reading progress indicator beyond "3 / 9" (spine index, not percentage or page number). No way to jump to a specific position.

**Impact:** High for keyboard-first users. Medium overall.

---

### Bookmarks and annotations
No bookmarks. No text highlighting. No notes. Users cannot mark important passages or return to specific locations other than the last-read position.

**Impact:** Medium. Required for study/research use cases; less critical for casual reading.

---

### Search
No in-book search.

**Impact:** Medium-high. Standard feature in all epub readers.

---

### Window and TOC state
Window size and position are not persisted across sessions. The TOC sidebar open/closed state resets on every launch.

**Impact:** Low. Noticeable friction but not blocking.

---

### External links
Links within epub content that point to external URLs (e.g. footnotes linking to web pages) are not handled. WKWebView will try to load them in the iframe; behaviour is undefined.

**Impact:** Low. Affects specific epub genres (academic texts, annotated editions).

---

### EPUB support completeness
- Fixed-layout EPUB (comic books, children's books, manga) is not supported. These render incorrectly or not at all.
- EPUB 3 media overlays (text-audio sync) are not supported.
- `<span>`-wrapped TOC labels are silently dropped (see code review #15).
- Epubs without a `<dc:identifier>` lose position state on re-open (see code review #8).

**Impact:** Varies. Fixed-layout is a significant gap if the app targets graphic content.

---

### Metadata and discovery
No book metadata display (cover, author, publisher, description). No library view. The window title shows the book title which is the only metadata surface.

**Impact:** Low for single-book use. Medium if multiple windows are open simultaneously — windows are indistinguishable from the Dock.

---

## Priority recommendations

| Priority | Feature | Rationale |
|---|---|---|
| 1 | Dark/light theme + reader CSS injection | Immediate comfort improvement; dark shell with white content is jarring |
| 2 | Font size controls | Accessibility baseline |
| 3 | Keyboard navigation (arrow keys, space) | Expected by all desktop users |
| 4 | Recent files / re-open from within window | Reduces friction for switching books |
| 5 | Bookmarks | Enables study/research use case |
| 6 | In-book search | Standard feature |
| 7 | Persist window size + TOC state | Polish |
| 8 | Cover image in window title bar / Dock | Distinguishability with multiple windows |
| 9 | External link handling | Forward to default browser |
| 10 | Fixed-layout EPUB detection and message | Honest failure over silent misrender |

---

## What is working well

- Render quality is high — native WKWebView means real-world epubs display as intended by their authors.
- Position persistence is reliable.
- File association works correctly with macOS conventions.
- Multi-window works — each book in its own window.
- The TOC subsection tracking (added in phase 2) is a differentiator vs. simpler readers.
- Build is clean and the codebase is well-structured for continued development.

---

## Suggested phase 3 scope

**Reading comfort sprint:** dark mode content injection (inject a CSS override into the iframe based on system appearance), font size controls (inject CSS `font-size` override), and keyboard navigation. These three together produce a usable daily-driver experience.

**Discovery sprint:** recent files list (persistent, shown when no book is open), re-open last book on launch, window title includes author name.
